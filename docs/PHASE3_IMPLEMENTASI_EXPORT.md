## Ringkasan

Phase 3 menambah kemampuan **download dan export** ke Invoice Maker. User kini bisa mendownload invoice sebagai PDF profesional, mengekspor seluruh daftar invoice ke Excel, dan mengekspor detail invoice ke CSV. Semua file digenerate **server-side** di backend Go.

**Pencapaian Utama**: Invoice Maker kini benar-benar bisa dipakai untuk membuat dan mengirim invoice ke client — bukan sekadar viewer.

---

## Keputusan Arsitektur

### 1. Server-Side Generation

Semua file (PDF, Excel, CSV) digenerate di **backend Go**, bukan di frontend browser.

**Alasan:**
- Output konsisten di semua browser/device (tidak tergantung rendering engine)
- File size lebih kecil (binary PDF langsung dari library)
- Frontend tetap ringan — tidak perlu dependency PDF/Excel library
- Sesuai learning goal (belajar Go libraries: `fpdf`, `excelize`)

**Flow:**
```
Frontend → klik Download
  → fetch GET /api/invoices/:id/pdf (dengan JWT token)
  → Backend ambil invoice + items dari DB
  → generateInvoicePDF() → return []byte
  → Gin return Content-Type: application/pdf + Content-Disposition: attachment
  → Frontend terima Blob → trigger <a> download
```

### 2. Library Selection

| Kebutuhan | Library | Alasan |
|---|---|---|
| PDF | `github.com/go-pdf/fpdf` v0.9.0 | Maintained fork of `gofpdf`, API stabil, tidak perlu dependency system (font built-in) |
| Excel | `github.com/xuri/excelize/v2` v2.10.1 | De facto standard Go Excel library, support formatting, auto-filter, multiple sheets |
| CSV | `encoding/csv` (stdlib) | Simpel, tidak perlu library eksternal |

### 3. Route Ordering (Gin Gotcha)

Endpoint `/api/invoices/export/excel` **harus didaftarkan sebelum** `/api/invoices/:id`. Kalau tidak, Gin akan menangkap `export` sebagai value parameter `:id` dan mengembalikan 404.

Route order final:
```
GET  /api/invoices              # list all
GET  /api/invoices/export/excel # export (SEBELUM :id!)
GET  /api/invoices/:id          # get single
POST /api/invoices              # create
PUT  /api/invoices/:id          # update
DELETE /api/invoices/:id        # delete
GET  /api/invoices/:id/pdf      # download PDF
GET  /api/invoices/:id/csv      # download CSV
```

### 4. Frontend Download Pattern

File binary di-fetch dengan JWT token, dikonversi ke Blob, lalu di-trigger download via temporary `<a>` element:

```typescript
// export.ts
const blob = await response.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url;
a.download = filename;
a.click();
// cleanup
URL.revokeObjectURL(url);
```

#### `backend/export.go`

Dua fungsi export:

**Excel** — `generateInvoicesExcel(invoices []Invoice) ([]byte, error)`:
- Sheet "Invoices" dengan 7 kolom: Invoice ID, Client, Date, Items, Tax Rate, Total Amount, Created At
- Header row: bold putih di atas bg biru
- Auto-filter di semua kolom
- Format mata uang di kolom Total Amount
- Column widths disesuaikan
- Auto-filter untuk sorting/filtering

**CSV** — `generateInvoiceCSV(inv Invoice) ([]byte, error)`:
- Header: Invoice ID, Client, Date, Tax Rate
- Items table: Description, Qty, Unit Price, Amount
- Totals section: Subtotal, Tax, Total
- Menggunakan `encoding/csv` (stdlib)

### Endpoint API Baru

Semua endpoint di-protect oleh middleware `authenticate()`.

#### `GET /api/invoices/export/excel`

Export semua invoice user saat ini ke Excel (.xlsx).

```bash
curl -X GET http://localhost:8080/api/invoices/export/excel \
  -H "Authorization: Bearer eyJhbGc..." \
  -o invoices.xlsx
```

Response: `Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`

#### `GET /api/invoices/:id/pdf`

Download single invoice sebagai PDF profesional.

```bash
curl -X GET http://localhost:8080/api/invoices/{id}/pdf \
  -H "Authorization: Bearer eyJhbGc..." \
  -o invoice.pdf
```

Response: `Content-Type: application/pdf`

#### `GET /api/invoices/:id/csv`

Download single invoice sebagai CSV.

```bash
curl -X GET http://localhost:8080/api/invoices/{id}/csv \
  -H "Authorization: Bearer eyJhbGc..." \
  -o invoice.csv
```

Response: `Content-Type: text/csv`

## Masalah yang Ditemui & Diperbaiki

### 1. Route Conflict Gin: `/export/excel` vs `/:id`

**Gejala**: Semua export endpoint return 404 meski terdaftar di Gin debug output.

**Root Cause**: `GET /api/invoices/export/excel` didaftarkan setelah `GET /api/invoices/:id`. Gin menangkap `export` sebagai nilai `:id` dan mencari invoice dengan ID "export" — tidak ditemukan → 404.

**Fix**: Pindahkan `GET /api/invoices/export/excel` **sebelum** `GET /api/invoices/:id`. Gin memprioritaskan exact match di atas parameterized route kalau urutannya benar.

**Lesson**: Route dengan path literal harus didaftarkan sebelum route dengan parameter di path yang sama. Ini berlaku untuk semua HTTP router (Gin, Echo, Chi, etc).

### 2. Port Already In Use (Testing)

**Gejala**: Setelah edit kode, backend tetap melayani request dengan route lama.

**Root Cause**: Proses backend sebelumnya masih berjalan di background, proses baru gagal bind ke port 8080 (silent fail — hanya muncul di log `[ERROR] listen tcp :8080: bind: address already in use`).

**Fix**: Gunakan `fuser -k 8080/tcp` untuk kill proses sebelum restart.

**Lesson**: Saat develop, pastikan proses sebelumnya benar-benar mati. Gunakan `fuser -k` atau `lsof -ti:8080 | xargs kill`.

### 3. fpdf.PdfWriterBuffer Tidak Ada

**Gejala**: Kompilasi error: `undefined: fpdf.PdfWriterBuffer`

**Root Cause**: API fpdf menggunakan `bytes.Buffer` (interface `io.Writer`), bukan custom buffer type.

**Fix**: `var buf bytes.Buffer; pdf.Output(&buf)`.

### 4. excelize AutoFilter API Signature

**Gejala**: `too many arguments in call to f.AutoFilter`

**Root Cause**: `AutoFilter` menerima 3 argumen: `(sheet, ref string, opts)`, dimana `ref` adalah Excel range string seperti `"A1:G10"`.

**Fix**: Gunakan `excelize.CoordinatesToCellName` untuk generate cell reference, lalu gabungkan dengan `fmt.Sprintf`.
